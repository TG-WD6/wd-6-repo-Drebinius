Opdracht 1

CSS kan included worden op 3 verschillende manieren.

door middel van:
  • een external stylesheet
  • een internal stylesheet
  • inline styling

external

Een external stylesheet wordt over het algemeen het meest gebruikt. Hierin wordt CSS in een apart .css bestand opgeslagen. Vervolgens kun je deze aan je HTML linken dmv 'link rel="stylesheet" href="styles.css"' Een voordeel van deze wijze is dat je 1 .css file aan meerdere HTML's kan linken, waardoor dezelfde CSS op verschillende HTML applied kan worden.

internal

Een internal stylesheet bevindt zich in het HTML doc zelf. Dit is CSS geplaatst in een style element dat zich in de HTML head bevindt. Een nadeel van deze wijze is dat de CSS alleen deze HTML file beinvloedt. Dit zou dus betekenen dat als je een site-wide aanpassing wilt doorvoeren, alle inhoud van de style elements in de HTML bestanden stuk voor stuk aangepast zouden moeten worden.

inline

Inline styling zijn toevoegingen die in een enkel HTML element geplaatst worden, achter een 'style="#"' attribuut. Dit betekent dat voor algemene stijlaanpassing er per element aangepast moet worden. Ook zorgt het voor een minder overzichtelijke code omdat HTML en CSS niet gescheiden worden gehouden.
Er zijn wel uitzonderingen waarin inline styling van toepassing kan zijn. Als je met je huidige edit rights geen content buiten bepaalde elements mag aanpassen, kan je dit toch doen door inline toe te passen.

Opdracht 2

een CSS ruleset bestaat uit 3 onderdelen:

  • Selectors
  • Properties
  • Values

Elke CSS start met een of meerdere selector(s). De selector target specifieke secties HTML om te beinvloeden.

Vervolgens komen de properties aan bod. Dit zijn identifiers die duidelijk aangeven welke feature ze influencen, bijv font-size, background-color of border.

Hierna volgen de values, dit zijn de waarden die je aan properties toeschrijft. bijv blue aan color

Als een property een value toegewezen heeft gekregen, noem je deze samen een CSS declaration.
Alle CSS declarations van 1 selector tesamen noem je een CSS declaration block.
Al deze components samengevoegd vormen de uiteindelijke CSS ruleset.

Opdracht 3

De 4 soorten selectors die we gaan beschrijven:
  • Type selectors
  • Class selectors
  • ID selectors
  • Descendant selectors

Type selectors

een type selector, ook wel tag name selector of element selector genoemd, specificeert een bepaalde HTML tag/element in je HTML doc.

Example * {
font-family: 'Lucida Sans', 'Lucida Sans Regular', 'Lucida Grande',
'Lucida Sans Unicode', Geneva, Verdana, sans-serif;
}

span {
background-color: yellow;
}

strong {
color: rebeccapurple;
}

em {
color: rebeccapurple;
}
In het bovenstaande voorbeeld zijn *, span, strong en em de type selectors.

Class selectors

De class selector begint met een punt (.) en daarop volgend de naam van de class die je hebt toegeschreven.
Dus als je een <p> de class="kaas" geeft, kan je dit element aanschrijven dmv de class selector .kaas te gebruiken.

Example .kaas {
text-decoration-color: hotpink;
}
In het bovenstaande voorbeeld staat er een punt ( . ) voor de class kaas, wat alle elementen met die aangegeven class selecteert.

De class selector begint met een punt (.) en daarop volgend de naam van de class die je hebt toegeschreven.
Dus als je een <p> de class="kaas" geeft, kan je dit element aanschrijven dmv de class selector .kaas te gebruiken.

Example .kaas {
